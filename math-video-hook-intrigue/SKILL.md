# 数学動画の語り口 — じらし・裏切り型（おもしろトピック寄り）

## このスキルの役割

数学が苦手な人向けの解説動画で、**視聴者を没頭させる「語り口・字幕文言」**を生成する。対象は、**一般にあまり知られていない／直感に反する「おもしろトピック寄り」**の題材。ゴールは「へえ！そうなの！？」を作ること。

一般的な数学解説は淡々とただ説明するだけになりがちで、それが「つまらない・冷たい」の正体。このスキルは、学習科学で効果が確認された仕掛けと、この作り手固有の"色"を組み合わせて、淡々を抜け出す。

> このスキルは「語り」のレイヤーだけを担当する。動画全体の作り方（1本1気づき、具体→半具体→式、動きと言葉の同期、強調は1画面1箇所、色は主役1＋補助1など）はプロジェクトの CLAUDE.md に従うこと。矛盾したら CLAUDE.md を優先。

## 共通の語りのDNA（この作り手の色）

姉妹スキルと共通の8項目。**これは毎回全部入れる義務リストではなく、この作り手の"引き出し"の一覧。** 1本の動画に全部盛ると型が透けて「またこれか」と飽きられる。**1本あたり、題材に最も効く3〜4個を選んで使い、残りは温存する**。**なぜ効くか**もセットで理解して使うこと。

1. **面倒さ・つまずきへの共感を丁寧に入れる。**「わかります、僕も最初そうでした」と語り手が横に並ぶ。→ 数学不安を持つ人は、語り手が同じ気持ちを言葉にしてくれると身構えが解ける。淡々とした解説はこの同伴が抜けているから冷たい。
2. **視聴者のセリフを代弁する。**「知らねえよ！って人もいれば、なんとなくやってる人もいますよね」と複数タイプを名指しで拾う。→「あ、自分のことだ」と刺さる幅が広がる。
3. **従来法・ダメなやり方の弱点を、感情でなく論理で示す。**「面倒」で終わらせず「なぜダメか」を理由で言う。→ 動機づけがつながり、納得して次に進める。
4. **ダメなパターンには強めに言い切る。**「それは今すぐやめましょう」「その覚え方は損します」とはっきり断言する。→ 優しいだけでなく、要所で言い切ることがこの作り手の芯。ぼかさない。**言い切るときは"未来の失敗の予告"で接地するのが定番**：「その覚え方だと、ちょっと難しい文章問題になった瞬間"どこに当てはめれば？"と迷子になりますよ」と、視聴者が近い将来ぶつかる具体的な困りごとを描く。→ 抽象的に「ダメ」と切るより説得力が出て、視聴者が自分の未来として想像できる。「ダメ」＋「なぜなら、こう詰むから」をセットにする。**ポジ版もある**：「複雑な問題ほど、この切り分けが超大事になってきます」と、これから効いてくる重要性を予告する形。ダメ出しの場面ではネガ版（詰みますよ）、単なる混同・理解不足の場面ではポジ版（効いてきますよ）、と使い分ける。
5. **難所には例えを入れる。**日常（料理・買い物・ゲーム）を土台に、図で見せられるものは図で示す前提の言葉にし、時にバカげた・大げさな例えも辞さない。→ 抽象は、身近な具体に翻訳された瞬間に腑に落ちる。**バカげた例えを使うときの標準手順は「日常で着地→極端へ振る→セルフツッコミで戻る」のエスカレート構造**：まず完全に日常の光景で着地させ（「ピザは4等分か8等分にして食べますよね」）、そこから段階的に極端へ振り（「では頭のネジを一旦外し、狂ったように100等分してみると…」）、直後に自分でツッコんで真面目に戻る（「見るも無惨な姿ですが、この奇行の裏には（真面目に）大切な意味が潜んでいます」）。日常が先にあるから極端化してもついてこられる。可能なら、極端化そのものが数学的本質（無限に細かく＝極限 など）と一致する例えを選ぶと、ふざけた見た目の裏に本物を仕込めて最強。**攻めた言葉はマイルド化する**：強い表現を使いたい場面では、意味は保ちつつ叩かれにくい言い回しに置き換える（「頭のネジを外す」「奇行」あたりが落とし所）。
6. **効能・ゴールを具体的に予告する。**「この動画を見ると、A ができて、B の意味もわかって、C まで届きます」と得られるものを並べる。→ 見続ける動機になる。**バリエーションとして「立場の逆転」も強い**：「授業では何の説明もなく出されたこの公式を、10分後には"逆に説明できる"ようになっているはず」のように、教わる側→説明できる側への入れ替わり＋具体的な所要時間で到達点を描く。
7. **口調はですます＋時々タメ口。ユーモアは毎回1〜3回、配置を決めて入れる。**基本は丁寧で、決め台詞や共感の瞬間だけタメ口を混ぜて距離を詰める。**ユーモア（エスカレート例え、セルフツッコミ、軽いボケ）は毎回の動画に少なくとも1回入れる。配置は、視聴者が離れやすい序盤と、説明に飽きてきそうな中盤〜終盤。上限は3回**。くどくしない。笑いは狙いすぎず、要所に一撃ずつ。
8. **解決策は「たった◯つの」に集約して差し出す。**「図の裏に隠れている、たった一つの"ルール"」「たった1つのポイントを押さえれば使い分けが手に取るようにわかる」のように、押さえるべき核を少数に絞って予告する。→ 覚える量が少ないと示すことで「それだけでいいの？」という軽さ（＝簡単さの予告と同じ効き方）が生まれ、1本1気づきの原則とも噛み合う。**ただし数はその題材の実態に合わせる。** 本当は2つあるのに「たった1つ」と言うために歪めない。1つなら「たった1つ」、2つなら「2つだけ」と正直に絞る。

## このスキル固有の掴み設計 — 隠して引っ張る

姉妹スキルとの違いはここだけ。**おもしろトピックは「知らない」ことが武器**なので、答えや正体をすぐ明かさず、じらして引っ張る。

- **答えを先に言わない。** 冒頭は「問い」か「直感を裏切る主張」だけを置き、正体（定理名・手法名・結論）は伏せる。→ 好奇心は「知りたいこと」と「今知っていること」の隙間から生まれる。隙間を開けたまま本編に引き込む。
- **わざと予想を裏切る。**「常識ではこう。でも実は逆なんです」を入口にする。→ 人は自分の予想が外れた瞬間、正しい情報を強く記憶する。意外性は記憶にも効く。
- **開いたループを最後に回収する。**「その正体は、最後に種明かしします」と予告し、動画の最後で伏線を回収する。→ 人は未完了のことを気にして覚え続ける。最後まで引っ張れる。

つまりこのスキルの掴みは **「共感 → 直感を裏切る主張 → 正体は伏せてじらす → 効能予告」** の順。

## オープニングの基本形（土台であって、毎回の義務ではない）

```
1. 問いかけ or 身近な入口（数式では始めない）
2. 視聴者の予想・直感を代弁する（「たぶんこう思いますよね」）
3. それを裏切る主張を、正体は伏せて置く（「でも実は…」）
4. 面倒さ・戸惑いへの共感を一言（「わかります」）
5. 効能予告＋じらし（「最後に種明かしします」で本編へ）
```

これは迷ったときに戻る土台。**5要素すべてを毎回入れる必要はなく、順番の入れ替え・省略・別の入口への差し替えは自由。**

## マンネリ回避の原則（重要）

毎回同じ骨格で作ると、視聴者に「またこれか」と型を見抜かれ、じらしも裏切りも効かなくなる。連続で作るときは次を守る。

- **入口を回す。** 問いかけ／日常の光景／クイズ／いきなり裏切る主張から始める。この中から選び、**直前の動画と同じ入口を使わない**ことを基本にする。
- **裏切りの置き場所も変えてよい。** 毎回冒頭で裏切ると「どうせ最初に"でも実は"が来る」と読まれる。中盤まで泳がせてから裏切る回があってもいい。
- **型は「効くから使う」のであって「型だから使う」のではない。** どのDNA・技法も、その題材に効かないなら外す。
- 唯一崩さないのは土台の思想だけ：**数式や定義から淡々と始めない／視聴者を置き去りにしない／嘘をつかない。** ここさえ守れば、あとの型はぜんぶ道具。

## 具体例（オープニング）

**題材：0.999… は 1 と等しい**

> 「0.999… って、1 と同じ数だと思いますか？」
>
> たぶん多くの人が「いやいや、1 よりちょっとだけ小さいでしょ」って思いますよね。0.9 の後に 9 がどれだけ続いても、1 にはあと一歩届かない感じがする。わかります、その感覚、めちゃくちゃ自然です。
>
> でも実は、0.999… は 1 とぴったり同じなんです。近いとかじゃなくて、完全にイコール。
>
> 「そんなバカな」って思った人ほど、この動画は面白いはずです。理由は3つの見方で説明していくんですが、いちばんスッキリするやつは最後にとっておきますね。

仕掛けの内訳：1行目で**問いかけて隙間を開ける**（答えを言わない）。2行目で**視聴者の直感を代弁＋共感**。3行目で**予想を裏切る主張**を、証明の正体は伏せて置く。最後で**効能予告（3つの見方）＋じらし（一番いいのは最後）**で本編へ。

## 本編〜締めの口調ルール

- **専門用語は使う瞬間に必ず言い換えを添える。**「極限、ようは、どこまでも近づいていった先のことですね」のように。厳密さは図や式の側で担保し、言葉は易しく保つ。
- **裏切りの瞬間に"間"を作る。** 種明かしの直後は、字幕なら決め台詞を短く独立させて画面に残す。詰め込まない。
- **共感の再挿入。** 難所の直前で「ここ、みんな一回止まるところです」と先回りして戸惑いを言語化する。
- **締めは「一言でいうと」。** その回の気づきを1文に圧縮し、冒頭で開いたループを回収して閉じる。

---

## 禁止事項（この作り手として避けること）

- **淡々と定義から入る。** 「〜とは、〜である」で始めるのは禁止。必ず問いか身近な入口から。
- **正体を冒頭で全部明かす。** このスキルはじらしが命。手法名・結論を1行目で言わない。
- **共感なしで正論だけ言う。** 「間違いです」と切って捨てるだけは冷たい。まず代弁と共感、そのうえで言い切る。
- **例えなしで抽象を押し通す。** 難所に日常の例え・図の言葉がないまま進めない。
- **同じ文を字幕と音声で丸ごと重複させる。** 目と耳で二重に読ませると負荷が上がる。どちらかを"語り"の本体にし、もう一方はキーワードだけにする。

## 生成前の最終チェック

**位置づけ：これは「全部にチェックを付けるための義務リスト」ではなく、引き出しの棚卸し。** 使っていない項目があるのはむしろ健全（全部盛りは型バレのもと）。

**土台（これだけは毎回）**
- [ ] 数式・定義から淡々と始めていないか
- [ ] 視聴者を置き去りにしていないか（共感・代弁・救いのどれかが、どこかにあるか）
- [ ] 嘘をついていないか（嘘の自己開示・実態と違う「たった1つ」）
- [ ] ユーモアを1〜3回、序盤 or 中盤〜終盤の飽き所に配置したか（くどくないか）
- [ ] 直前の動画と同じ入口・同じ骨格になっていないか（マンネリ回避）
- [ ] ですます＋時々タメ口になっているか

**使った型の品質確認（使ったものだけ見る）**
- [ ] じらし → 正体を伏せたループを、締めで必ず回収したか（開きっぱなしにしない）
- [ ] 裏切り → 視聴者の予想を先に代弁してから裏切ったか
- [ ] 落として上げる → 詰めは軽く一回、救いはすぐ隣か
- [ ] 「たった◯つの」集約 → 数は題材の実態に正直か
- [ ] エスカレート例え → 日常着地→極端→セルフツッコミの順か。攻めた言葉はマイルド化したか
- [ ] 言い切り → 未来予告で接地したか（詰みますよ／効いてきますよ）
- [ ] 自己開示 → 本当に自分が知らなかった題材か
